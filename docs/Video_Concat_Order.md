# 视频合并顺序规范

## 1. 概述

本文档详细说明 Coiner 中视频合并的顺序逻辑，包括场景级别和目标视频级别的视频段合并规则。

## 2. 视频合并层次结构

### 2.1 核心概念

- **场景（Scene）**：视频的基本组成单位，每个场景包含一个或多个视频片段
- **视频片段（Video Clip）**：单个视频文件或其部分内容
- **片头视频（Intro Video）**：视频开头的特殊视频片段
- **普通视频段（Regular Video Clips）**：场景中的常规视频片段
- **静默前缀（Silence Prefix）**：视频开头的静默静止帧
- **场景字幕预烧录（Scene Subtitle Pre-Burning）**：在合并前将字幕烧录到每个场景视频中

### 2.2 合并层次

1. **场景内部合并**：处理单个场景内的视频片段
2. **场景字幕预烧录**：将字幕嵌入到每个场景视频中（可选）
3. **场景间合并**：按场景顺序合并所有场景（包含静默前缀）
4. **最终合成**：添加 pillarbox、标题、BGM、响度归一化等

## 3. 视频连接模式（VideoConcatMode）

### 3.1 定义

视频连接模式控制**场景内部**视频片段的连接方式，不影响场景间的顺序。

### 3.2 可用模式

- **顺序模式（sequential）**：按照视频片段的原始顺序连接
- **随机模式（random）**：随机打乱场景内的视频片段顺序

### 3.3 作用范围

- **全局配置**：视频连接模式是全局统一配置，适用于所有场景
- **仅影响**：场景内部的普通视频片段
- **不影响**：
  - 场景间的顺序
  - 最终合成顺序
  - 字幕合成

## 4. 合并顺序规则

### 4.1 最终合成顺序

1. **静默前缀**：视频开头的静默静止帧（从第一个场景的第一帧提取）
2. **场景视频**：按照场景顺序依次排列
3. **场景内视频**：根据视频连接模式处理

### 4.2 场景级别处理

对于每个场景：

1. 提取场景中的所有视频片段
2. 根据视频连接模式处理片段顺序：
   - 顺序模式：保持原始顺序
   - 随机模式：打乱片段顺序
3. 处理视频片段（调整大小、应用效果等）
4. 合并为场景级视频（包含音频）
5. **字幕预烧录**（可选）：将字幕烧录到场景视频中，音频直接复制

### 4.3 场景间合并处理

1. **创建静默前缀视频**（如果配置了静默时长）
   - 使用第一个场景的第一帧作为画面（无字幕）
   - 使用微弱粉红噪声作为音频（防止 AAC 极端压缩）
   - 所有参数与场景视频保持一致（帧率、采样率、分辨率等）

2. **字幕预烧录**（如果启用了字幕）
   - 逐个场景烧录字幕
   - 音频 stream-copy，不重新编码
   - 视频编码参数与原始场景一致，确保 stream-copy 合并安全

3. **构建合并列表**
   - 静默前缀（第一个）
   - 场景 1 视频
   - 场景 2 视频
   - ...

4. **FFmpeg stream-copy 合并**（快速路径）
   - 使用 `concat` demuxer + `-c copy`
   - 所有场景参数一致时，零重编码，速度极快
   - 失败时自动回退到 MoviePy 重编码

### 4.4 目标视频级别处理

1. 合并后的视频输入（已包含静默前缀+所有场景）
2. 添加 pillarbox（3:4 → 9:16）
3. 添加标题
4. 添加 BGM（与原始音频混合）
5. 响度归一化（EBU R128, -16 LUFS）
6. 最终编码输出

### 4.5 音频处理原则

- **场景音频**：直接使用场景视频中的原始音频
- **静默前缀音频**：使用极低音量粉红噪声（~-66dB）而非纯静默，确保播放器时序正常
- **背景音频**：如果添加背景音频，基本原则是音频适配视频时长
- **同步处理**：字幕预先烧录到场景视频中，确保音字幕完美同步
- **响度归一化**：在最终编码阶段统一进行，避免多次归一化

### 4.6 字幕处理原则

- **处理时机**：场景级别，合并前预烧录（而非最终合成阶段）
- **同步保证**：字幕与音频在同一场景内生成，时间轴完全一致
- **样式控制**：支持自定义字幕样式、位置和大小
- **失败回退**：单个场景字幕烧录失败时，回退到无字幕版本，不影响整体

## 5. 实现差距分析

### 5.1 当前实现问题

1. **场景概念缺失**：当前实现没有明确的场景级别概念，所有普通视频片段被视为一个整体
2. **顺序控制不当**：随机模式会打乱所有视频片段，破坏场景顺序
3. **连接模式作用范围**：连接模式影响了整个视频集，而不仅仅是场景内部

### 5.2 期望实现

1. **场景级别处理**：按场景分组处理视频片段
2. **保持场景顺序**：场景间严格按照原始顺序排列
3. **连接模式限制**：连接模式仅影响场景内部的视频片段顺序
4. **音频处理优化**：确保音频与视频同步，避免音频混乱

### 5.3 实现逻辑细节

根据当前实现，视频合成的核心逻辑如下：

1. **音频时长计算**：首先计算音频时长或视频总时长
2. **视频处理**：处理每个视频文件，分割成子剪辑并应用效果
3. **视频合并**：按顺序拼接处理后的视频剪辑
4. **音频合并**：添加音频（外部音频或场景视频的原始音频）
5. **最终输出**：生成最终视频文件

当前实现遵循音频适配视频的原则，确保音频与视频保持同步。

## 6. 技术实现

### 6.1 数据结构调整

- 引入场景级别的数据结构，将视频片段按场景分组
- 为每个场景存储其视频片段和连接模式
- 场景视频路径支持动态切换（原版 / 带字幕版）

### 6.2 合并逻辑优化

1. **场景预处理**：
   - 按场景分组视频片段
   - 对每个场景应用连接模式

2. **字幕预烧录**（可选）：
   - 检查场景是否有字幕文件
   - 调用 `burn_subtitles_to_scene_video` 烧录字幕
   - 成功则使用带字幕的版本，失败回退到原版

3. **静默前缀创建**：
   - 使用 ffprobe 分析第一个场景的音频参数
   - 使用相同参数创建静默前缀（帧率、采样率、声道数等）
   - 使用微弱粉红噪声替代纯静默

4. **场景间合并**：
   - 优先使用 FFmpeg stream-copy（快速路径）
   - 构建 concat list 文件
   - 执行 `-f concat -c copy`
   - 失败时回退到 MoviePy 重编码（慢路径）

5. **最终处理**：
   - pillarbox（3:4 → 9:16）
   - 标题叠加
   - BGM 混合
   - 响度归一化（EBU R128）
   - 最终编码

### 6.3 关键函数

- **`burn_subtitles_to_scene_video`**：场景级字幕烧录（audio stream-copy）
- **`create_silence_prefix_video`**：创建静默前缀视频（参数一致 + 粉红噪声）
- **`analyze_audio_params`**：分析视频音频参数（ffprobe）
- **`concat_videos_stream_copy`**：FFmpeg stream-copy 快速合并
- **`combine_all_scenes`**：多场景合并主函数（快速路径 + 回退）
- **`process_final_video`**：最终视频处理（支持跳过字幕）

### 6.4 代码模块

- **场景级别处理**：按场景分组处理视频片段
- **连接模式应用**：确保全局统一的连接模式仅应用于场景内部
- **字幕预烧录**：场景级字幕嵌入，确保音字幕同步
- **静默前缀管理**：参数一致性保证 stream-copy 成功率
- **快速路径合并**：优先 stream-copy，失败回退重编码

## 7. 示例流程

### 7.1 输入

- **全局视频连接模式**：随机（适用于所有场景）
- 场景 1：视频片段 A、B、C
- 场景 2：视频片段 D、E、F
- 音频：场景内语音（TTS 生成）
- 字幕：每个场景独立的 SRT 文件
- BGM：背景音乐（可选）
- 静默时长：3 秒

### 7.2 处理流程

**场景级别处理**：
1. 处理场景 1：
   - 应用全局连接模式（随机）：打乱视频片段为 B、A、C
   - 处理并合并为场景 1 视频（包含音频）
   - 字幕预烧录 → scene_01_with_subs.mp4
2. 处理场景 2：
   - 应用全局连接模式（随机）：打乱视频片段为 F、D、E
   - 处理并合并为场景 2 视频（包含音频）
   - 字幕预烧录 → scene_02_with_subs.mp4

**合并准备**：
3. 创建静默前缀视频（3 秒，从场景 1 第一帧提取，无字幕）
   - 参数：帧率 30fps、采样率与场景一致、粉红噪声音频

**场景间合并**：
4. 按顺序合并（FFmpeg stream-copy）：
   - 静默前缀 + 场景 1（带字幕） + 场景 2（带字幕）

**最终处理**：
5. 添加 pillarbox（3:4 → 9:16）
6. 添加标题
7. 添加 BGM（与原音频混合）
8. 响度归一化（-16 LUFS）
9. 最终编码输出

### 7.3 输出

最终视频顺序：
- [0:00-0:03] 静默前缀（第一帧静止画面，无字幕）
- [0:03-...] 场景 1（B → A → C，带字幕）
- [...-结束] 场景 2（F → D → E，带字幕）

## 8. 总结

视频合并顺序应遵循以下原则：

1. **保持场景顺序**：场景间严格按照原始顺序排列
2. **灵活处理场景内部**：通过连接模式控制场景内视频片段的顺序
3. **字幕预烧录**：场景级字幕嵌入，确保音字幕完美同步
4. **静默前缀先创建**：在字幕预烧录前创建，确保静默部分无字幕
5. **参数一致性**：静默前缀与场景视频参数完全一致，确保 stream-copy 成功
6. **快速路径优先**：优先使用 FFmpeg stream-copy，失败回退重编码
7. **音频适配视频**：音频处理应适配视频时长，确保同步

通过正确实现这些规则，可以确保视频合成结果符合预期，同时保持场景的逻辑连贯性。

## 9. Roadmap

### 9.1 未来功能规划

- **目标视频级别的片头**：在目标视频级别添加片头功能，允许用户为整个视频添加统一的片头
- **场景级别的音频控制**：为每个场景添加独立的音频处理选项
- **智能音频同步**：优化音频与视频的同步处理，避免音频混乱
- **高级转场效果**：为场景间添加更丰富的转场效果，同时确保音频平滑过渡
- **多轨道字幕**：支持多语言字幕轨道切换
- **场景级 BGM 切换**：不同场景可配置不同的背景音乐
